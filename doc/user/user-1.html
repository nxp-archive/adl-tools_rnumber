1.1 Large Integers

RNumber was initially developed to provide support for large integers in C++ for developing an application to generate random stimulus for microprocessor validation (Raptor). This application domain has some unique requirements that were not present in number libraries available at the time. These main requirement not available was to be able to control whether the number would expand the number of bits that represent it on overflow or to wrap around to zero on overflow. Since expansion of number on overflow is not typically used in the problem domain RNumber was developed for, it is not as convenient to use as wrapping on overflow.

Be advised. RNumber has had extensive use in applications using fixed numbers. dynamic RNumber code has had very little use.

Since signed integers were not required for the original application they are not supported. In other ways RNumber are as convenient as built in types for C++

Some time after the original use of RNumber in Raptor, bindings were created to use them with Bigloo for an application to analyze Raptor output (figleaf). In order to deal with the name mangling and overloaded operators, a C interface was created. To my knowledge, the C interface has never been used directly. Only the portions of the RNumber interface necessary to write figleaf were written. However, this is most of the useful portions and the rest could be fairly easily added.

Shortly after using RNumber with bigloo, a request was made to provide perl bindings as well. The C interface was used to create a simple swig interface.

1.3 Build Utility and Examples

RNumber includes a build utility called rnumber-config that may be used to aid in constructing command lines for building. Running rnumber-config with no options will show the supported command line flags. Look at the comments at the top of the examples in rnumber/examples to see how to make good use of this command.

There is a simple example for each of C++, C, perl, and Bigloo scheme in the rnumber/examples directory. There are comments at the top of each that explains how to compile or run the example.
