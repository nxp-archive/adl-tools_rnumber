Variable Index
Concept Index

Only the exported functions can be called by bigloo. Currently none of the random number generation functions are exported.

4.1 Variables

There is only one rnumber variable, *rnumber-version*. This is the version of the library.


4.2 Memory Management

If Bigloo was built with finalization enabled, Bigloo will destroy rnumbers that were created by calling the bigloo rnumber construction function (rnumber-ctor arg).


4.3 Functions

Exported Functions
FunctionDescription
(rnumber? a)Returns true if a is an rnumber.
(rn? a)Alias for rnumber?.
(rnumber-ctor arg)Constructs a new rnumber based on the argument. The argument can be null, an integer, a string, or an rnumber.
(rn-ctor arg)Alias for rnumber-ctor
(rnumber->cstr rnumber . radix-prefix)This converts an rnumber into its string representation. The radix-prefix are optional arguments. The first optional argument is the radix to use, the second optional argument is whether to include a radix prefix at the begining of the string. For example "0x" for a hex number.
(rn->string rnumber . radix-prefix)Alias for rnumber->cstr
(rnumber-parse-string str)This parses a string representing a legal number in decimal, hex, or binary into an rnumber. If the string does not represent an rnumber, rnumber-parse-string returns #f
(rnumber->int rnumber)Convert an rnumber to a regular integer. No warning, error, or return code will indicate loss of data.
(rn->integer rnumber)Alias for rnumber->int
(rn=? a b)Compares two variables that represent either rnumbers or integers to see if they hold the same integer value.
(rn!=? a b)Returns (not (rn=? a b))
(rn<? a b)Compares two variables that represent either rnumbers or integers to see if the one represented by a is less than the one represented by b.
(rn<=? a b)Compares two variables that represent either rnumbers or integers to see if the one represented by a is less than or equal to the one represented by b.
(rn>? a b)Compares two variables that represent either rnumbers or integers to see if the one represented by a is greater than the one represented by b.
(rn>=? a b)Compares two variables that represent either rnumbers or integers to see if the one represented by a is greater than or equal to the one represented by b.
(rn<< a b)Left shift a by b bits. a should be an rnumber. b can be either an integer or an rnumber.
(rn>> a b)Right shift a by b bits. a should be an rnumber. b can be either an integer or an rnumber.
(rn- a b)Subtract b from a. b and a can be either integers or rnumbers. The return value is an rnumber unless both a and b are integers.
(rn+ a b)Add a to b. b and a can be either integers or rnumbers. The return value is an rnumber unless both a and b are integers.
(rn* a b)Mutliply a by b. b and a can be either integers or rnumbers. The return value is an rnumber unless both a and b are integers.
(rn-bitand a b)Return logical bit and of a and b. Either a or a may be an integer or an rnumber.
(rn-bitxor a b)Return logical bit xor of a and b. Either a or a may be an integer or an rnumber.
(rn-bitnot a)Return logical bit not of a. a may be an integer or an rnumber.
(rn-bitor a b)Return logical bit or of a and b. Either a or b must be an rnumber. Both can not be integers.
rn-sign-extend! rn bitSign extends the rnumber rn from the bit given by bit. rn must be an RNumber and bit must be an integer. bit is the big-endian bit from which to sign-extend. This function returns #t if there are no errors.
rn-get-bit rn bitReturns an integer of the bit-value in the rnumber at bit bit. rn must be an RNumber, bit must be an integer. This function indexes into the RNumber with big-endian notation (ie. bit 0 is most significant)
rn-get-bit-lsb rn bitReturns an integer of the bit-value in the rnumber at bit bit. rn must be an RNumber, bit must be an integer. This function indexes into the RNumber with little-endian notation (ie. bit 0 is least significant)
rn-set-bit! rn bit valueSets the bit in the rnumber at bit to the value of value. rn must be an RNumber, bit and value must be integers. bit indexes into the RNumber with bit-endian notation (ie. bit 0 is most significant). This function returns #t if there are no errors.
rn-set-bit-lsb! rn bit valueSets the bit in the rnumber at bit to the value of value. rn must be an RNumber, bit and value must be integers. bit indexes into the RNumber with little-endian notation (ie. bit 0 is least significant). This function returns #t if there are no errors.
rn-resize! rn new-sizeResizes the number to be new-size bits. rn must be an RNumber and new-size must be an integer. The function returns new-size if there are no errors.
rn-size rnReturns an integer representation of the bit-width of the RNumber. rn must be an RNumber.
rn-word-count rnReturns an integer representation of the number of words of memory with RNumber uses to store its number. rn must be an RNumber.
